Fast path for TPC-C new_order: bump d_next_o_id on one district row.

Uses index lookup + row lock + heap/WAL update only. Skips index_registry write because d_next_o_id is not part of idx_district_wd keys. Returns Ok(None) when the predicate is not an exact index key (caller should use update_rows_by_equalities).
