# Сравнение бенчмарков RustDB и PostgreSQL

## Обзор

Документ содержит результаты бенчмарков RustDB и типичные показатели PostgreSQL для ориентировочного сравнения. Учитывайте, что тесты различаются по методике и окружению.

---

## Результаты RustDB

### Базовые операции (benchmarks)

| Операция           | Время (среднее) |
|--------------------|-----------------|
| database_creation  | ~43 ps          |
| database_open      | ~20 ps          |

### SQL парсинг (sql_benchmarks)

| Операция           | parse_only | parse_and_plan | full_pipeline |
|--------------------|------------|----------------|---------------|
| SELECT simple      | ~7.5 µs    | ~8.2 µs       | ~10.6 µs      |
| SELECT WHERE       | ~10.3 µs   | ~10.3 µs      | ~17.0 µs      |

| UPDATE             | ~9.2 µs    | ~8.3 µs       | ~10.5 µs      |
| DELETE             | ~7.8 µs    | ~7.0 µs       | ~8.2 µs       |
| CREATE TABLE       | ~10.3 µs   | —              | —             |

### E2E (полный цикл: парсинг → планирование → выполнение → WAL)

| Тест                    | Время (среднее) |
|-------------------------|-----------------|
| insert_with_tx_wal      | ~26 ms/операция |
| inserts_with_tx_wal (5) | ~19 ms/батч    |
| begin_insert_commit     | ~26 ms/цикл    |
| select_full_scan_100    | ~70 µs          |

### Batch операции

| Тест                    | Время (среднее) |
|-------------------------|-----------------|

### I/O (io_benchmarks)

## Типичные результаты PostgreSQL

### pgbench (стандартный TPC-B‑подобный)

| Конфигурация              | TPS (транзакций/с) |
|---------------------------|--------------------|
| 16 клиентов, 4 worker     | ~2,900–3,000       |
| 800 клиентов, scale 100   | ~30,000–125,000*   |
| 1536 клиентов (PostgreSQL 15.6+) | ~300,000 |

\* Зависит от железа (CPU, диски, RAM).

### Ориентировочные значения

- **Простой INSERT**: ~1–5 µs на парсинг
- **Полный INSERT с WAL**: ~100–500 µs
- **Полный INSERT с fsync**: ~1–10 ms
- **Один запрос**: ~30–100 µs (без учёта сетевой задержки)

## Запуск бенчмарков

### RustDB

```bash
# Все бенчмарки
cargo bench

# Отдельные наборы
cargo bench --bench benchmarks
cargo bench --bench sql_benchmarks
cargo bench --bench e2e_benchmarks
cargo bench --bench io_benchmarks
cargo bench --bench performance_benchmarks
```

### PostgreSQL (pgbench)

```bash
# Инициализация
pgbench -i -s 10 pgbench_db

# Простой TPC-B‑подобный тест (60 s)
pgbench -c 16 -j 4 -T 60 pgbench_db

# С кастомным скриптом
pgbench -c 16 -j 4 -T 60 -f custom_script.sql pgbench_db
```

Пример `custom_script.sql` для сравнения с RustDB:

```sql
INSERT INTO bench_table (id, name, age) VALUES (1, 'Alice', 30);
```

## Сравнение

| Метрика                     | RustDB          | PostgreSQL (примерно) |
|----------------------------|-----------------|------------------------|
| Парсинг SQL                | ~7–10 µs        | ~1–5 µs                |
| E2E INSERT (1 запись)       | ~26 ms          | ~0.1–1 ms              |
| TPS (транзакций/с)         | ~38 (1/26ms)    | ~3,000–300,000         |
| I/O read/write page         | ~0.8 ms         | ~0.1–1 ms (зависит от ОС) |

### Выводы

1. **RustDB** — учебная/экспериментальная СУБД, фокус на архитектуре и SQL‑пайплайне.
2. **PostgreSQL** — зрелая СУБД с оптимизациями для высокой нагрузки и масштабирования.
3. **Разрыв по TPS** связан с:
   - отсутствием сетевого протокола;
   - синхронным fsync;
   - отсутствием пула соединений и параллельных воркеров;
   - ограничениями текущей реализации (страницы, WAL).

4. **Парсинг SQL** в RustDB сопоставим по порядку величины с PostgreSQL (микросекунды).
5. **E2E‑операции** в RustDB медленнее из‑за дискового I/O и WAL‑логирования.

- [ ] Параллельное выполнение запросов
